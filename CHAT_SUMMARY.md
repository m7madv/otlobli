# ملخص شامل — تطبيق otlobli (وسيط SHEIN لسوريا)

**المشروع:** `C:\Users\MOHAMMAD\Projects\SHEIN IN SIRYA`
**الفكرة:** تطبيق Capacitor (Android) بيفتح موقع SHEIN داخل InAppBrowser (مكتبة `@capgo/capacitor-inappbrowser`)، وبيحقن سكريبت JS (`src/services/sheinBrowserScript.ts`) يتحكم بالصفحة، يلتقط بيانات المنتج، ويضيفه لسلة otlobli الخاصة (مش سلة SHEIN). الكود الرئيسي للتطبيق بـ `src/App.tsx`.

---

## 1. مشاكل الـ layout (محلولة)

- **تشويه أسفل الصفحة الرئيسية لـ SHEIN** (كاروسيل عروض متراكب): سببه `isPresentAfterPageLoad: true` كان يخلي SHEIN يحمّل وهو مخفي (عرضه = 0)، فالكاروسيل يحسب أبعاده غلط. الحل: إرسال `__resize` message للـwebview لحظة ما يصير ظاهر (`InAppBrowser.postMessage`)، والسكريبت يستقبلها ويعمل `window.dispatchEvent(new Event('resize'))` + يعيد فحص فوري.
- **الـwebview يغطي/يبتعد عن شريط otlobli السفلي:** الحساب الصحيح بقى `navRect.top - bottomGap` (`bottomGap = 40`)، بدل `windowHeight - navHeight` (كان غير دقيق بسبب status bar/مناطق النظام).
- **هيدر otlobli ("otlobli" + جرس) كان ظاهر فوق SHEIN دايماً:** الحل: إخفاؤه بالكامل لما `sheinReady` تصير true (بـ`App.tsx` بالقسم الافتراضي لـ`renderScreen`).
- **بانرات SHEIN الثابتة بالأسفل (شحن مجاني..) كانت تتراكب مع شريط otlobli:** `hideStrayFixedBottomBars()` — تفحص نقاط محددة بأسفل الشاشة وتمشي لفوق لحد أصغر عنصر ثابت/مقاس فيه، وتخفيه. **مهم:** لا تستخدم `querySelectorAll('a,button')` بدون تحديد حجم — هاد بالضبط سبب كسر كبير قبل (غطّى الهيدر كامل بالغلط).

## 2. إعادة تصميم زر "أضف للسلة" (محلول)

- بدّلنا الزر العريض بالأسفل لزر **مربّع صغير ثابت بأعلى يسار الصفحة** (`#otlobli-cart-btn`)، بمكان أيقونة سلة SHEIN، 42×42px، خلفية خضراء، أيقونة 🛍.
- **سلوكه:**
  - لو بصفحة منتج + المقاس/اللون محددين (أو غير موجودين بالأساس) → يضيف المنتج (`addToCartFlow`).
  - لو لسا ناقص مقاس/لون → رسالة تنبيه.
  - لو **مش** بصفحة منتج → يفتح سلة otlobli (`requestOpenOtlobliCart` → postMessage `{type:'openCart'}` → `App.tsx` يستقبلها ويعمل `setScreen('cart')`).
- **Badge عداد** أحمر دائري فوق الزر، بيتحدّث حسب `cartItems.length` (عبر `__cartCount` message من React)، مع انيميشن "pop" (`otlobli-badge-pop`).
- **سلة SHEIN نفسها ممنوعة كلياً:** أي رابط/أيقونة سلة (`isCartLink`) بيتم اعتراض الكليك عليها + إعادة توجيه أي تنقل لرابط سلة (`blockCartNavigation`)، وكلهم بيوجّهوا لـ`requestOpenOtlobliCart()`.
- **إخفاء بصري لأيقونات السلة الحقيقية:** `hideSheinCartIcons()` (جديدة، آخر تحديث) — بتفحص نقاط بالهيدر فوق وبشريط الإجراءات تحت (جنب "أضف للسلة"/"اشتري الآن")، وتخفي أي عنصر صغير الحجم (icon-sized) يطابق `isCartLink`. **لم يتم التأكد منها على صفحة منتج فعلية بعد** — لازم تحقق.
- **إخفاء أيقونات الهيدر الزائدة** (قلب، إيميل، منيو) إلا البحث: `hideExtraHeaderIcons()` — نفس أسلوب فحص النقاط، بتستثني أي عنصر فيه `<input>` أو كلمة "search/بحث/camera".

⚠️ **قاعدة أمان مهمة تعلمناها بالتجربة الصعبة:** أي دالة "إخفاء" يجب أن:
1. تفحص **نقاط محددة** (`elementFromPoint`) وتمشي لفوق لأقرب عنصر قابل للنقر، **لا** تستخدم `querySelectorAll` عريض على كل الصفحة.
2. تتحقق إنه العنصر **صغير حجم أيقونة** (`width/height < 64-80px`) قبل الإخفاء — منعاً لإخفاء حاوية كبيرة بالغلط.

## 3. التقاط بيانات المنتج (اسم/سعر/صورة/لون/مقاس)

### الاسم ✅ شغال
- ترتيب المحاولة: JSON-LD (نوع `Product` فقط — SHEIN ما عندها هذا النوع غالباً، بس تركناه للمستقبل) → `og:title` (لو مش نص عام) → `h1`/كلاسات تخمينية → `document.title` (**فقط** كحل أخير بعد استنفاد كل المحاولات).
- `looksGenericTitle()`: ترفض نص مثل "ملابس نسائية ورجالية، تسوق الموضة على الموقع | شي إن" (تاغ لاين الموقع العام، مش اسم منتج).

### السعر ✅ شغال
- JSON-LD offers → meta `product:price:amount` → كلاس `[class*="price" i]`.

### المقاس ✅ شغال
- `findOptionContainer('size', ['المقاس','Size'])` — بحث بالكلاس، وfallback بالنص لو ما لقى.

### اللون (نص) ✅ شغال لمعظم المنتجات
- `getAttrLabelValue` — بيدوّر عن نص حرفي "اللون: ..." (أدق طريقة، لا تعتمد على تخمين كلاسات).
- **ملاحظة:** بعض المنتجات (مثل حزام بلون واحد) **ما فيها أصلاً color picker** — فـ"capturedColor" فاضي يكون طبيعي بهاي الحالة، مش بَق.

### أيقونة اللون (صورة الـ swatch) ⚠️ غير مؤكدة 100%
- `getSelectedColorSwatchImage` + `isSelectedSwatchEl` (يفحص aria-selected/checked/pressed، كلاس "selected/active/.."، أو `input.checked` فعلي).
- محتاج تجربة مؤكدة على منتج فيه ألوان متعددة فعلياً لتأكيد الدقة 100%.

### الصورة الرئيسية 🔧 شبه محلولة، بقي خلل أخير
**رحلة الإصلاح (مهم تفهمها لأي تعديل قادم):**
1. *اكتشاف:* صور SHEIN تستخدم **lazy loading** — الـ`src` فيها صورة placeholder عامة (`bg-logo-...png`)، والصورة الحقيقية بـ`data-src`. → `realImgSrc()` يتحقق من `data-src` أولاً (كان هذا صحيح من البداية لحسن الحظ).
2. *اكتشاف:* روابط الصور أحياناً **بدون بروتوكول** (`//img...` بدل `https://...`) — ينكسر لما يستخدم بسياق تطبيقنا. → `normalizeImageUrl()` يضيف `https:` تلقائياً.
3. *باگ كبير:* `isInPromoWidget()` (دالة استثناء بانرات "ثبّت التطبيق") كانت تحتوي كلمة `banner` عامة جداً، فطلعت بتستثني **كل صور معرض المنتج الحقيقية** (تأكدنا 100% عبر `adb logcat` إنه `promo=true` كان يطلع على كل الصور الحقيقية). تم تضييق الفلتر لـ `app-download|download-app|applink|app-banner|guide-popup` فقط. **هذا حلّ مشكلة "ولا صورة أبداً" نهائياً.**
4. *باگ تجميع المعرض:* `getGalleryImage()` كانت تجمّع الصور حسب العنصر الجد (grandparent) **كعنصر DOM فريد** — وبما إن كل صورة بمعرض SHEIN ملفوفة بعنصر `<li>` خاص بيها، كل صورة طلعت "مجموعة من واحد" فالدالة فشلت تلاقي مجموعة 3+. الحل: التجميع حسب **نص الكلاس** (`className` string) للعنصر الأب مباشرة (مش هوية العنصر) — هذا نجح ولقى المجموعة الصحيحة.
5. **المشكلة الحالية المفتوحة:** بعد إيجاد المجموعة الصحيحة، الدالة كانت تاخد أول عنصر بترتيب DOM (`items[0]`) — لكن هذا غير موثوق لأن مكتبات الكاروسيل (Swiper-like) ممكن تعيد ترتيب/تكرر عناصر DOM داخلياً. جرّبنا الإصلاح: اختيار الصورة **الأكبر حجماً فعلياً على الشاشة** (`getBoundingClientRect`) بدل أول بالترتيب — **لسا ما نجح بالكامل**، المستخدم بلّغ إنه بيجيب صورة #2 من 7 بدل #1 (الصورة الرئيسية/الغلاف).
   - **آخر خطوة:** ضفنا تشخيص مفصّل (`OTLOBLI_DBG M` و `OTLOBLI_DBG N0..N11`) يطبع لكل صورة بالمجموعة: العرض، الارتفاع، الموضع (top/left)، ونهاية الرابط. **لم يتم تحليل هذه البيانات بعد** — هذا أول شي يلي لازم تعمله بالشات الجديد: خلي المستخدم يضيف منتج فيه معرض صور، اسحب الـlogcat، وحلل لماذا الصورة التانية بتفوز على الأولى بالمساحة.

## 4. تحسينات تجربة المستخدم (محلولة)

- **إعادة محاولة تلقائية (Retry):** `addToCartFlow` يحاول لمدة ~5 ثواني (10 محاولات × 500ms) قبل ما يستسلم على أي حقل ناقص — لأن SHEIN صفحة ثقيلة وبعض البيانات بتلحق تتحمّل بعد الضغط على الزر مباشرة.
- **شاشة تحميل احترافية (Overlay):** تجمّد التفاعل، تعرض صورة/اسم/لون/مقاس + سبينر، وتتحدث حياً مع كل محاولة. لما تخلص، تتحول لـ"✓ تمت الإضافة".
- **شيبات تشخيص مرئية بالـ overlay** (✓/✗ لكل حقل: اسم، صورة، أيقونة اللون، سعر، لون، مقاس) — مفيدة لو المستخدم صوّر الشاشة وقت الضغط.
- **سطر واحد لاسم المنتج بفاتورة "تفاصيل التكلفة":** CSS (`.price-line span { overflow:hidden; white-space:nowrap; text-overflow:ellipsis; min-width:0 }`).

## 5. أداة تشخيص أهم من السكرين شوت

السكريبت فيه `debugSnapshot()` (يُستدعى تلقائياً أول ما تضغط زر "أضف للسلة") بيطبع بيانات حقيقية بـ`console.log`، وAndroid WebView بيمررها لـ`logcat` تحت تاغ `chromium`. **هذا أسرع وأدق من طلب سكرين شوت من المستخدم** — تقدر تسحبها مباشرة:

```powershell
$adb = "$env:LOCALAPPDATA\Android\Sdk\platform-tools\adb.exe"
& $adb -s emulator-5554 logcat -d -t 3000 | Select-String "OTLOBLI_DBG"
```

**مهم:** logcat بيقطّع الرسائل الطويلة جداً، فلازم كل سطر `console.log` يكون قصير (تحت ~150 حرف). لو احتجت بيانات JSON كبيرة، قسمها لأسطر منفصلة قصيرة، أو (أفضل) سجّل كل قيمة بسطر مستقل بدل JSON واحد كبير.

## 6. خط أنابيب البناء والنشر (Build/Deploy)

```powershell
cd "C:\Users\MOHAMMAD\Projects\SHEIN IN SIRYA"

npx cap sync android

$env:JAVA_HOME = "C:\Program Files\Android\Android Studio\jbr"   # ضروري! جافا النظام الافتراضية (17) غير متوافقة، Capacitor يحتاج 21

.\gradlew.bat assembleDebug

$adb = "$env:LOCALAPPDATA\Android\Sdk\platform-tools\adb.exe"
$apk = "C:\Users\MOHAMMAD\Projects\SHEIN IN SIRYA\android\app\build\outputs\apk\debug\app-debug.apk"
& $adb -s emulator-5554 install -r $apk
& $adb -s emulator-5554 shell am force-stop com.otlobli.app
& $adb -s emulator-5554 shell am start -n "com.otlobli.app/.MainActivity"
```

**قبل كل رفع، تحقق من صحة الـ JS المحقون** (لأن `sheinBrowserScript.ts` عبارة عن template literal فيه escaping بالباكسلاش `\\`، وغلطة فيه بتكسر السكريبت بصمت):

```bash
node -e "
const fs = require('fs');
const content = fs.readFileSync('src/services/sheinBrowserScript.ts', 'utf8');
const start = content.indexOf('\`'); const end = content.lastIndexOf('\`');
let script = content.slice(start+1, end).split('\\\\\\\\').join('\\\\');
fs.writeFileSync('check.js', script);
"
node --check check.js && echo OK && rm check.js
```

**المحاكي:** `emulator-5554`، فيه جلسة دخول محفوظة. **لا تستخدم `adb shell pm clear com.otlobli.app`** إلا للضرورة — بيمسح الجلسة ويرجّع لشاشة تسجيل الدخول (تسجيل دخول وضع تجريبي محلي: أي رقم + أي رمز تحقق بيشتغل).

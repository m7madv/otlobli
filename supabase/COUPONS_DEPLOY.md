# نشر نظام كود الخصم + إنشاء الأكواد

## 1) نشر الخلفية (مرة واحدة)
شغّل `supabase/schema.sql` كاملاً على قاعدة بياناتك (SQL Editor في Supabase).
الملف idempotent — آمن إعادة تشغيله. يُنشئ:
- جدول `coupons` (تعريف الأكواد)
- جدول `coupon_redemptions` (سجل الاستخدام، فريد لكل هاتف/جهاز)
- الدالة `redeem_coupon(...)` (تحقّق + حساب + استهلاك ذرّي)

قبل النشر: **الخصم لا يظهر إطلاقاً في التطبيق** (تدهور آمن — لا يتأثر أي مبلغ).

## 2) إنشاء كود خصم (من SQL Editor مباشرةً — بلا لوحة إدارة)

```sql
-- خصم نسبة مئوية 10% على كل المتاجر:
insert into public.coupons (code, kind, value, applies_to)
values ('WELCOME10', 'percent', 10, 'all');

-- خصم مبلغ ثابت 5$ على تيمو فقط:
insert into public.coupons (code, kind, value, applies_to)
values ('TEMU5', 'fixed', 5, 'temu');

-- خصم 15% على شي إن، حد أدنى للطلب، صلاحية وحد استخدام إجمالي:
insert into public.coupons (code, kind, value, applies_to, min_subtotal_syp, max_uses, expires_at)
values ('SHEIN15', 'percent', 15, 'shein', 500000, 200, '2026-12-31');
```

الحقول:
- `kind`: `percent` (value = 1..100) أو `fixed` (value = مبلغ بالدولار، يُحوَّل لليرة بسعر `usd_to_syp_rate`).
- `applies_to`: `all` / `shein` / `temu`.
- `max_uses` (اختياري): سقف إجمالي لكل الأكواد. `min_subtotal_syp` (اختياري): حد أدنى.
- `starts_at` / `expires_at` (اختياري).

## 3) منع الاحتيال (مضمون بالخلفية)
- **استخدام واحد لكل رقم هاتف** لكل كود (فهرس فريد) — لا يُعاد بتغيير الحساب.
- **استخدام واحد لكل جهاز** (معرّف محفوظ محلياً) — طبقة إضافية.
- الاستهلاك **ذرّي** داخل الدالة: يستحيل استخدام الكود مرتين حتى مع طلبين متزامنين.

## قيود معروفة (للتحسين لاحقاً)
- معرّف الجهاز حالياً من localStorage (يُعاد ضبطه عند حذف بيانات التطبيق/إعادة
  التثبيت). للربط الأقوى على مستوى العتاد: إضافة `@capacitor/device`. القيد
  الأساسي (رقم الهاتف) يبقى فعّالاً في كل الأحوال.
- الكود يُستهلك لحظة "تطبيق" الكود في شاشة الدفع؛ لو غادر الزبون بلا إتمام
  الطلب يُعتبر مستخدماً. (تصميم متعمّد لمنع إعادة الاستخدام.)

## اختبار سريع بعد النشر
1. أنشئ كوداً تجريبياً (أعلاه).
2. بالتطبيق: أضف منتجاً → الدفع → أدخل الكود → «تطبيق» → يظهر سطر الخصم ويقل الإجمالي.
3. جرّب نفس الكود ثانيةً بنفس الرقم → «لقد استخدمت هذا الكود من قبل».

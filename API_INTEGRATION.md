# طلبية API Integration Point

وصل التطبيق الآن إلى طبقة ربط جاهزة للاستبدال من تنفيذ محلي إلى APIs حقيقية.

## الخدمات المطلوبة قبل الإنتاج

- WhatsApp OTP: إرسال رمز تحقق وتأكيده وإرجاع جلسة مستخدم.
- SHEIN product lookup: قراءة رابط منتج وإرجاع الصور والألوان والمقاسات والسعر والوزن.
- B2B Sham Cash: قراءة التحويلات الواردة ومطابقة التحويل بالمبلغ الدقيق.
- Al Qadmous tracking: حفظ رقم القدموس وجلب حالة الشحنة.
- Backend database: حفظ المستخدمين والعناوين والسلة والطلبات والمدفوعات وسجل الحالات.

## مكان الربط في الكود

- العقود موجودة في `src/services/appApi.ts`.
- التنفيذ المحلي المؤقت موجود في `src/services/localAppApi.ts`.
- تكامل Supabase موجود في `src/services/supabaseAppApi.ts`.
- اختيار الخدمة يتم من `src/services/index.ts`: إذا كانت مفاتيح Supabase موجودة يستخدم Supabase، وإلا يرجع للوضع المحلي.
- API لوحة الإدارة موجود في `api/admin/orders.ts` ويحتاج `SUPABASE_SERVICE_ROLE_KEY` و `ADMIN_PIN` داخل Vercel فقط.
- عند توفر APIs حقيقية، أنشئ ملف adapter جديد مثل `src/services/httpAppApi.ts` بنفس واجهة `TalabiehApi`، ثم بدّل الاستيراد داخل `src/App.tsx`.

## قاعدة مطابقة شام كاش الحالية

المطابقة تعتمد على المبلغ الدقيق فقط:

```txt
المهم جداً: ادفع نفس المبلغ بالضبط حتى تتم المطابقة تلقائياً.
```

لا نعتمد حالياً على رقم الطلب داخل ملاحظة الدفع، ولا على حساب المرسل.
